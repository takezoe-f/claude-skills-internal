# Claude Skills Internal

個人用のClaude Codeスキルコレクション。ロール系スキルと自律実行スキルをまとめたリポジトリ。

## スキル一覧

| スキル | 説明 |
|---|---|
| designer | UI/UXデザイナーとして情報設計・ワイヤーフレーム・UIデザイン・アクセシビリティ対応を行う |
| dev-engineer | シニア開発エンジニアとして設計・実装・レビュー・リファクタリング・パフォーマンス最適化を行う |
| pdm | プロダクトマネージャーとして要件定義・ユーザーストーリー・PRD作成・優先順位付けを行う |
| qa-engineer | QAエンジニアとしてテスト戦略策定・テストケース設計・バグ分析・テスト自動化を行う |
| team-coordinator | チームコーディネーターとして各ロールを横断的に統括しプロジェクト全体を推進する |
| autonomous-executor | ユーザー不在時にタスクを自律的に完遂する無人実行モード |

## インストール

```bash
git clone https://github.com/takezoe-f/claude-skills-internal.git /tmp/claude-skills-internal
# 各スキルをシンボリックリンクで配置
for skill in designer dev-engineer pdm qa-engineer team-coordinator autonomous-executor; do
  ln -s /tmp/claude-skills-internal/$skill ~/.claude/skills/$skill
done
```

## 各スキルの詳細

### designer

UI/UXデザイナーとして、ユーザー体験の質を最大化するデザイン専門家。情報設計・ワイヤーフレーム・UIデザイン・デザインシステム・ユーザビリティ評価・アクセシビリティ対応を行う。

**主な機能:**
- デザイン思考フレームワークに基づくUI/UX設計
- ワイヤーフレーム・プロトタイプ作成
- WCAG 2.1 AA準拠のアクセシビリティ対応
- デザインレビュー
- レスポンシブ設計（Mobile Firstアプローチ）

**トリガーフレーズ:** `デザイナーとして` `UI/UX視点で` `デザインレビューして` `ワイヤーフレームを作って` `画面設計して` `/designer`

---

### dev-engineer

シニア開発エンジニアとして、技術的卓越性を追求する。コードの品質・保守性・セキュリティ・テスタビリティを最優先とする技術的エキスパート。

**主な機能:**
- アーキテクチャ設計・技術選定
- TDDベースの実装
- セキュリティ・パフォーマンスを意識したコーディング
- コードレビュー（Critical/Major/Minor/Nit分類）
- リファクタリング・パフォーマンス最適化

**トリガーフレーズ:** `エンジニアとして` `実装して` `コードレビューして` `アーキテクチャを設計して` `リファクタリングして` `/dev-engineer`

---

### pdm

プロダクトマネージャーとして、ユーザー価値とビジネス価値を最大化する意思決定を行う。要件定義からロードマップ策定まで、プロダクト開発の上流工程を担当。

**主な機能:**
- 要件定義・ユーザーストーリー作成
- PRD（プロダクト要件定義書）作成
- RICE/MoSCoWによる優先順位付け
- KPI設計・KPIツリー策定
- MVP定義・スコープ管理

**トリガーフレーズ:** `PdMとして` `要件定義して` `ユーザーストーリーを書いて` `PRDを作って` `優先順位を決めて` `/pdm`

---

### qa-engineer

QAエンジニアとして、プロダクト品質を守り抜く。テスト戦略策定からテスト自動化まで、品質保証の全プロセスを担当。

**主な機能:**
- テスト戦略策定（テストピラミッドベース）
- テストケース設計（境界値・状態遷移・組み合わせ等）
- バグレポート作成
- テスト自動化（Arrange-Act-Assertパターン）
- 探索的テスト
- 品質レビュー

**トリガーフレーズ:** `QAとして` `テスト設計して` `テストケースを作って` `品質チェックして` `エッジケースを洗い出して` `/qa-engineer`

---

### team-coordinator

チームコーディネーター/テックリードとして、エンジニア・PdM・デザイナー・QAの各ロールを横断的に統括する。タスク分解・ロール割り当て・進捗管理・リスク管理・合意形成を行い、チーム全体の生産性を最大化する。

**主な機能:**
- マルチロール分析（各専門家視点での並行分析）
- プロジェクト立ち上げ・タスク分解・ロール割り当て
- 進捗管理（クリティカルパス・ブロッカー把握）
- 横断レビュー統合（各ロール間の矛盾調整）
- Go/No-Goリリース判定

**トリガーフレーズ:** `チームとして進めて` `全体をまとめて` `タスクを分解して各ロールに振って` `横断的にレビューして` `マルチロールで分析して` `/team-coordinator`

---

### autonomous-executor

ユーザー不在時にタスクを自律的に完遂する無人実行モード。夜間や離席中のバッチ的な開発作業に最適。

**主な機能:**
- タスク分解と順次自律実行
- エラー発生時の自動リカバリー（3回リトライ後スキップ）
- 完了レポート（`AUTONOMOUS_REPORT.md`）自動生成
- プロンプトインジェクション対策
- セキュリティガードレール（`.env`変更禁止、force push禁止等）

**トリガーフレーズ:** `自律実行` `夜間実行` `確認なしで` `寝てる間に` `自動で進めて` `おまかせ` `/auto-run`
